Tab-Struktur: Welche Tabs soll die App haben?

Bottom Tabs sind gesetzt — aber welche Aufteilung passt am besten zu deinem Workflow?

A

3 Tabs: Dashboard · Kontakte · Profil

Dashboard zeigt Übersicht + letzte Transaktionen. Transaktionen werden immer über einen Kontakt erstellt (Kontakt → Details → Transaktion). Minimalistisch.

B

4 Tabs: Dashboard · Kontakte · Transaktionen · Profil

Transaktionen als eigener Tab mit Chronik, Filter, Suche. Transaktion erstellen geht sowohl über Kontakt als auch über den Transaktionen-Tab. Wie in der aktuellen App.

C

3 Tabs + FAB: Kontakte · Transaktionen · Profil

Kein Dashboard-Tab, stattdessen ein Floating Action Button (FAB) für Quick-Actions (neue Transaktion, neuer Kontakt). Die wichtigsten Zahlen stehen im Header oder als Karten über der Kontaktliste.
